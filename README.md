# 🧠 Semi-Supervised Spatio-Temporal Transformer Modeling for Facial Action Unit Detection

SSSTTM for multi-label facial AU detection on in-the-wild video data.

![SSSTTM Pipeline](Figures/Figure%201.png)

## ✨ At a Glance

- Task: multi-label Facial Action Unit detection
- Core idea: spatio-temporal CNN-Transformer modeling + audio fusion + semi-supervised Mean Teacher consistency learning
- Dataset target: Aff-Wild2 style data pipeline

## 🗂️ Repository Layout

- `models/`: model definitions (`hrformer`, `tformer`, `avformer`, etc.)
- `dataloader/`: LMDB-backed dataset loading, transforms, split creation
- `metrics/`: AU/EX/VA metrics
- `teacher/`: semi-supervised Mean Teacher consistency losses and ramp schedules
- `112_align/`: LMDB creation/inspection scripts
- `train.py`, `train2.py`, `train3.py`: CNN/Transformer baseline training variants (different preset defaults)
- `train_mt.py`, `train_mt2.py`: separate Mean Teacher training variants
- `train_mtmm.py`: DDP-friendly Mean Teacher training script
- `train_all.py`: two-stage launcher (baseline stage then Mean Teacher stage)
- `test_aff2.py`, `test_aff2mm.py`: AU inference/evaluation scripts
- `show_loss.py`, `show_result.py`, `show_CAM.py`, `show_CAMmm.py`: analysis and visualization helpers
- `opts*.py`: preset argument configurations

## 🧩 Data Expectations

Key runtime arguments:

- `--root`: processed dataset root
- `--cache_dir`: split cache directory (`split_dict_*.pkl`)
- `--lmdb_label_dir`: LMDB labels/images directory

Typical required assets:

- extracted frame paths
- `video2orignal.pkl`
- LMDBs: `.croped_aligned_jpeg`, `.croped_aligned_mask` (optional), `.label_au`, `.label_expr`, `.label_va`

If split cache files do not exist, split metadata is generated by `dataloader/data_split.py`.

## 🧪 Training Modes and Model Choices

- Full two-stage training: stage-1 CNN/Transformer training plus semi-supervised Mean Teacher fine-tuning (`train_all.py`)
- Separate baseline experiments: `train.py`, `train2.py`, `train3.py` (different presets in `opts.py`, `opts2.py`, `opts3.py`)
- Separate Mean Teacher experiments: `train_mt.py`, `train_mt2.py`
- Multi-GPU Mean Teacher: `train_mtmm.py`

Main `--model_name` options used in this repo:

- `hrformer` (HRNet-based spatio-temporal backbone)
- `avformer` (audio-visual transformer path)
- `tformer`, `vggformer`, `vformer`
- `sformer`, `dsformer`
- `i3d`, `mc3d`
- `emonet`, `audio`

If a custom `--model_name` is not matched, training falls back to the baseline image model path in the training scripts.

## 🚀 Quick Start

Two-stage CNN/Transformer + Mean Teacher (recommended end-to-end):

```bash
python train_all.py \
  --stage1_exp_dir ./exp_dir/hrformer_au \
  --stage2_exp_dir ./exp_dir/hrformer_au_mt \
  --train_args "--root /path/to/aff2_processed --cache_dir /path/to/cached_data --lmdb_label_dir /path/to/LMDB" \
  --mt_args "--root /path/to/aff2_processed --cache_dir /path/to/cached_data --lmdb_label_dir /path/to/LMDB"
```

Separate baseline AU training:

```bash
python train.py \
  --task AU \
  --model_name hrformer \
  --modality A;V \
  --root /path/to/aff2_processed \
  --cache_dir /path/to/cached_data \
  --lmdb_label_dir /path/to/LMDB \
  --exp_dir ./experiments/super/hrformer_au
```

Separate Mean Teacher training:

```bash
python train_mt.py \
  --task AU \
  --model_name hrformer \
  --modality A;V \
  --root /path/to/aff2_processed \
  --cache_dir /path/to/cached_data \
  --lmdb_label_dir /path/to/LMDB \
  --exp_dir ./experiments/mt/hrformer_au_mt
```

Audio-visual vs visual-only example:

```bash
# Audio-visual run
python train.py --model_name avformer --modality A;V --task AU

# Visual-only run (omit audio by removing A from modality)
python train.py --model_name hrformer --modality V --task AU
```

Multi-GPU examples:

```bash
# Two-stage launcher with distributed mode
python train_all.py --ddp --devices 0,1,2,3 --hardcode_batch

# Direct DDP Mean Teacher training
torchrun --standalone --nnodes=1 --nproc_per_node=4 train_mtmm.py --task AU --model_name hrformer --modality A;V
```

Evaluation:

```bash
python test_aff2.py
```

## 📝 Practical Notes

1. Several scripts still contain hard-coded absolute paths from the original environment; always override with CLI arguments.
2. Some model files reference optional pretrained checkpoints; replace paths or disable pretraining if unavailable.
3. Inference scripts define `model_path` near the top; set it to your checkpoint before running.
4. `train.py` / `train2.py` / `train3.py` and `train_mt.py` / `train_mt2.py` are intentionally separate experiment presets; use the one matching your intended setup.
5. For multi-GPU Mean Teacher, prefer `train_mtmm.py` (DDP) or run `train_all.py` with `--ddp`.
